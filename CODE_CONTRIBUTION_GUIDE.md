# CODE_CONTRIBUTION_GUIDE.md: Standardized Contribution Flow

Welcome to the **AI Era Developer Roadmap** community! We are thrilled to have you contribute to the AI-Era Developer Roadmap. To maintain the quality and global accessibility of our project, please follow these guidelines.

---

## 🏗️ Contribution Standards

All contributions must strictly follow the standards defined in [STRUCTURE_DESC.md](../STRUCTURE_DESC.md). 

### Document Format
- **Language:** English only.
- **Content:** Technical, objective, and neutral. No political or ideological discussions.
- **Tone:** Educational and actionable.
---

## 🚀 How to Contribute

1. **Fork the Repository:** Create a personal copy of the project.
2. **Create a Branch:** `git checkout -b feature/your-topic-name`
3. **Add Content:** 
   - For simple topics, create a `.md` file in the appropriate category directory.
   - For complex topics, create a directory with a `README.md` and a `demos/` sub-folder.
4. **Commit and Push:** Ensure your commit message is clear and descriptive.
5. **Submit a Pull Request (PR):** 
   - Describe the value of your addition.
   - Confirm that you've followed the structural and language requirements.
---

## ⚖️ Code of Conduct

- **Be Respectful:** We are a global community.
- **No Self-Promotion:** Avoid including excessive personal links or promotional content.
- **Maintain Neutrality:** Focus on technology, not its socio-political implications.

---

## ❓ Need Help?

If you're unsure where your topic fits, feel free to open an **Issue** with the tag `proposal` to discuss your idea with the maintainers.

*Thank you for helping developers build the future!*
