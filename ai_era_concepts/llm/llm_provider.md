# LLM Providers

## Overview
LLM providers are services that offer access to Large Language Models (LLMs) via APIs or local environments. They manage the underlying infrastructure, model hosting, and inference scaling, enabling developers to build AI-powered applications without the need for complex GPU cluster management.

## Why it matters
In the AI era, LLM providers act as the "cognitive utilities" of the digital world. Choosing the right provider is critical for:
- **Model Diversity:** Accessing specialized models for coding, reasoning, or creative tasks.
- **Cost Efficiency:** Balancing performance with token-based pricing or open-source local hosting.
- **Scalability:** Leveraging cloud infrastructure to handle high-concurrency requests.
- **Data Privacy:** Ensuring compliance with regional regulations or keeping data local via on-premise solutions.

## Key Principles: Standard Protocols
To simplify integration, most LLM providers adhere to or provide compatibility layers for standardized API protocols. This interoperability allows developers to switch providers with minimal code changes.

### 1. OpenAI Chat Completions API
The de facto industry standard for stateless text generation.

#### Authentication
- **Header:** `Authorization: Bearer YOUR_API_KEY`

#### Request Schema (Simplified)
```json
{
  "model": "gpt-4o",
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"}
  ],
  "stream": false,
  "temperature": 0.7
}
```

**Key Attributes:**
- `model` (string, Required): The ID of the model to use (e.g., `gpt-4o`, `gpt-3.5-turbo`).
- `messages` (array, Required): A list of messages comprising the conversation.
  - `role` (string): The role of the messenger. Valid values: `system` (context setting), `user` (human), `assistant` (AI), `tool` (function results).
  - `content` (string): The actual text content of the message.
- `stream` (boolean, Optional): If `true`, tokens will be sent as data-only server-sent events as they become available. Default: `false`.
- `temperature` (number, Optional): Sampling temperature between 0 and 2. Higher values (e.g., 0.8) make output more random, lower values (e.g., 0.2) make it more focused and deterministic.

#### Response Schema (Simplified)
```json
{
  "id": "chatcmpl-123",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Hello! How can I help you today?"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "total_tokens": 21
  }
}
```

**Key Attributes:**
- `choices` (array): The list of model-generated completions.
  - `message`: The message object generated by the model.
  - `finish_reason` (string): Why the model stopped. Values: `stop` (natural end), `length` (max tokens hit), `tool_calls` (waiting for tool result).
- `usage` (object): Statistics for the request, including `prompt_tokens`, `completion_tokens`, and `total_tokens`.

#### Demos

<summary>cURL</summary>

```bash

  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```
</details>

)

### 2. Anthropic Messages API
Focused on "human-assistant" interaction and safety.

#### Authentication
- **Header:** `x-api-key: YOUR_API_KEY`
- **Header:** `anthropic-version: 2023-06-01` (Required)

#### Request Schema (Simplified)
```json
{
  "model": "claude-3-5-sonnet-20240620",
  "max_tokens": 1024,
  "messages": [
    {"role": "user", "content": "Hello, Claude!"}
  ]
}
```

**Key Attributes:**
- `model` (string, Required): Anthropic model ID (e.g., `claude-3-5-sonnet-20240620`).
- `max_tokens` (number, Required): The maximum number of tokens to generate before stopping.
- `messages` (array, Required): Input messages. Note: System prompts are passed via a top-level `system` attribute in Anthropic's API, not as a message role.
- `role` (string): Must be `user` or `assistant`.

#### Response Schema (Simplified)
```json
{
  "id": "msg_01Xv9Sdtv61vV3U7n2g6XfVb",
  "content": [{"type": "text", "text": "Hello, Claude!"}],
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 12,
    "output_tokens": 15
  }
}
```

**Key Attributes:**
- `content` (array): The output content blocks.
- `stop_reason` (string): Why generation stopped. Values: `end_turn` (natural end), `max_tokens`, `stop_sequence`, `tool_use`.

#### Demos
<details>
<summary>cURL</summary>

```bash
curl https://api.anthropic.com/v1/messages \
     --header "x-api-key: $ANTHROPIC_API_KEY" \
     --header "anthropic-version: 2023-06-01" \
     --header "content-type: application/json" \
     --data '{
         "model": "claude-3-5-sonnet-20240620",
         "max_tokens": 1024,
         "messages": [{"role": "user", "content": "Hello, Claude!"}]
     }'
```
</details>

<details>
<summary>Node.js</summary>

```javascript
import Anthropic from '@anthropic-ai/sdk';

  apiKey: process.env.ANTHROPIC_API_KEY,
});

const message = await anthropic.messages.create({
  model: "claude-3-5-sonnet-20240620",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello, Claude!" }],
});
console.log(message.content[0].text);
```
</details>

<details>
<summary>Python</summary>

```python

client = anthropic.Anthropic(api_key="your-api-key")

    model="claude-3-5-sonnet-20240620",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello, Claude!"}]
)
print(message.content[0].text)
```
</details>

### 3. Google Gemini API
Natively designed for multimodal inputs and long context windows.

#### Authentication
- **Header:** `x-goog-api-key: YOUR_API_KEY`
- **Query Param:** `?key=YOUR_API_KEY`

#### Request Schema (Simplified)
```json
{
  "contents": [{
    "parts": [{"text": "Write a story about a magic backpack."}]
  }]
}
```

**Key Attributes:**
- `contents` (array, Required): The content of the current conversation.
  - `role` (string, Optional): The role of the content. Values: `user`, `model`.
  - `parts` (array): Data parts. For Gemini, this can be `text`, `inline_data` (images/files), or `function_call`.

#### Response Schema (Simplified)
```json
{
  "candidates": [{
    "content": {
      "parts": [{"text": "Once upon a time, there was a magic backpack..."}],
      "role": "model"
    },
    "finishReason": "STOP"
  }],
  "usageMetadata": {
    "totalTokenCount": 44
  }
}
```

**Key Attributes:**
- `candidates` (array): Generated completion options.
- `finishReason` (string): Why the model stopped. Values: `STOP`, `MAX_TOKENS`, `SAFETY`, `RECITATION`.

#### Demos
<details>
<summary>cURL</summary>

```bash
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=$GEMINI_API_KEY" \
    -H 'Content-Type: application/json' \
    -X POST \
    -d '{
      "contents": [{
        "parts":[{"text": "Write a story about a magic backpack."}]
      }]
    }'
```
</details>

<details>
<summary>Node.js</summary>

```javascript
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
const model = genAI.getGenerativeModel({ model: "gemini-1.5-flash" });

const result = await model.generateContent("Write a story about a magic backpack.");
console.log(result.response.text());
```
</details>

<details>
<summary>Python</summary>

```python
import google.generativeai as genai
import os

genai.configure(api_key="your-api-key")
model = genai.GenerativeModel('gemini-1.5-flash')

response = model.generate_content("Write a story about a magic backpack.")
print(response.text)
```
</details>

## AI Context
Modern AI development often adopts a **Multi-Model Strategy**. By standardizing on the OpenAI compatible protocol, developers can use LLM Aggregators (like OpenRouter or 302.ai) to dynamically switch between models based on performance, cost, or availability. Additionally, local runners like Ollama provide a seamless path for private, offline development using the same API structures as enterprise cloud providers.
